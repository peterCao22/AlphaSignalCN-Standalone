## AlphaSignalCN-Standalone 导出清单（Manifest）

### 0. 约定
- 目标独立目录根：`AlphaSignalCN-Standalone/`
- 本清单里所有路径均以独立目录根为基准（例如 `docs/`、`scripts/`、`data/`）。
- **原则**：代码与文档优先；大体积数据默认不拷贝（可选拷贝“最小必要集”）。

### 1. 必须保留（核心运行最小集）
#### 1.1 代码
- `predict_stock.py`
- `scripts/pattern_matcher.py`
- `scripts/enhanced_features_service.py`
- `scripts/download_index_bar1d.py`
- `stockainews/`（预测依赖的 adapters/crawlers/services/core 等子模块）
- `scripts/update_market_sentiment.py`
- `scripts/update_dragon_seats.py`

#### 1.2 模型与配置
- `models/second_wave_lgb.model`
- `models/second_wave_xgb.model`
- `models/feature_names.json`
- `.env.example`（建议提供示例，不要带真实密钥）

#### 1.3 文档
- `docs/SHORT_TERM_RELAY_FEATURES_SPEC.md`
- `docs/PROJECT_ROADMAP_AND_RISKS.md`
- `docs/DATA_STORAGE_POSTGRES_PLAN.md`
- `docs/RUNBOOK_20260120.md`

### 2. 推荐保留（用于稳定运行/复盘）
#### 2.1 SQLite 数据库（小体积，建议保留）
- `data/market_sentiment.db`
- `data/dragon_seats.db`
- `data/historical_patterns.db`

#### 2.2 运行输出目录（可空）
- `data/predictions_trigger_*.json`（历史预测结果，用于去重与复盘）

### 3. 可选保留（按需）
#### 3.1 原始 CSV 数据（体积大，建议按需拷贝）
- `data/raw/kline/kline_all.csv`（大）
- `data/raw/chips_all.csv`（中）
- `data/raw/limit_up/dragon_list.csv`（中）
- `data/raw/index_bar1d.csv`（小，建议保留）
- `data/raw/*.csv`（增强特征：`hot_rank.csv`、`concept_component.csv`、`concept_bar.csv`、`auction_factors.csv`）
- `data/raw/features/features_kline_stock_character_*.csv`（大）

### 4. 必须忽略（不应导出）
- `venv/`（每台机器重新建）
- `__pycache__/`
- `*.log`、`logs/`
- `data/temp/`（可运行时自动生成）
- 任何包含密钥/Token 的真实 `.env`（只保留 `.env.example`）

### 5. 导出后检查清单（快速验收）
- 能运行单股预测：`python predict_stock.py --symbol 000620`
- 能读取 DB：`data/market_sentiment.db`、`data/dragon_seats.db` 存在且可查询
- 若要使用增强特征：`data/raw/hot_rank.csv` 等文件日期覆盖到目标 trigger_date
